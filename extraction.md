# 内容获取管线 v1.6

## 路由逻辑

根据 `progress.json` 的 `source_type` 字段路由到对应管线：

| source_type | 管线 | 说明 |
|-------------|------|------|
| `pdf` | 路径 A：PDF 提取 | 用户上传教材 PDF |
| `ppt` | 路径 A：PPT 提取 | 用户上传课件 PPT |
| `markdown` | 路径 M：Markdown 提取 | 用户上传教材 Markdown 文件 |
| `ai_generated` | 路径 B：AI 内容生成 | 用户仅输入科目名，AI 生成全部内容 |
| `course_library` | 路径 C：预置课程 | 从预置课程库加载（未来实现） |

---

## 路径 A：PDF/PPT 提取

### 教材 PDF 提取

```
1. pdftotext / pymupdf 提取全文
2. 若无可选中文本 → tesseract OCR（中文 chi_sim）
3. 提取 PDF 书签作为章节目录
4. 若无书签 → 字体大小 + "第X章" 模式推断
5. 逐页 OCR 提取（如需），按页面范围切分章节
```

### PPT 提取

```
1. .pptx → python-pptx 提取文本框+表格+备注
2. .ppt → LibreOffice headless 转 PDF → 提取文本
3. 按标题页/小结页分组幻灯片
```

### 知识点提取

```
1. 逐节扫描：识别定义、定理、公式、示例、注意提示
2. 重要性判定：
   - ⭐⭐⭐：在小结中 / 有推导 / 是后续前置
   - ⭐⭐：有定义块 / 有例题 / 在习题中
   - ⭐：背景介绍 / 拓展阅读
   - 📌：「注意」「常见误区」提示 / 易混淆概念对
3. 公式使用 LaTeX 语法
```

### 等级过滤

```
L1: 仅 ⭐⭐⭐
L2: ⭐⭐⭐ + 前置依赖的 ⭐⭐
L3: ⭐⭐⭐ + 全部 ⭐⭐ + 小结中的 ⭐
L4: 全部
```

## 路径 B：AI 内容生成

### 前置条件

- `progress.json` 中 `source_type = "ai_generated"`
- 已有 `subject` 和 `exam_type`

### B.1 阶段一：生成章节结构

基于模型的训练知识 + 可选 WebSearch，生成完整的章节目录。

#### B.1.1 默认知识库生成（即时，零搜索）

模型根据 `{科目名称} + {考试类型}` 从自身训练知识中生成章节结构。

生成规则：
1. 分析科目的标准课程体系（国内大学本科/研究生标准教材结构）
2. 输出 6-12 章的章节大纲
3. 每章包含：章节标题 + 3-5 个核心知识点关键词
4. 按教学逻辑排序（基础→进阶→综合）
5. 考试类型影响章节范围：
   - 期末：覆盖整门课的标准内容
   - 期中：覆盖前半学期的内容
   - 随堂：聚焦近期章节
   - 考研：覆盖更广更深，可能跨教材

示例（科目="通信原理"，考试类型="期末"）：

```
第1章 绪论 — 通信系统模型、信息度量、信道容量
第2章 信号与频谱 — 傅里叶分析、功率谱密度、信号带宽
第3章 模拟调制 — AM/DSB/SSB/FM 原理与抗噪性能
第4章 数字基带传输 — 码型设计、码间串扰、眼图
第5章 数字频带传输 — ASK/FSK/PSK/QAM 调制解调
第6章 信源编码 — 抽样定理、量化、PCM
第7章 信道编码 — 差错控制、线性分组码、卷积码
第8章 同步原理 — 载波同步、位同步、帧同步
```

#### B.1.2 WebSearch 增强（可选）

若用户选择使用 WebSearch 增强，执行以下查询策略：

**查询模板**（按优先级）：
1. `"{科目全称} 课程大纲 site:edu.cn"` — 获取真实教学大纲
2. `"{科目全称} 教学大纲"` — 获取中文教学大纲
3. `"{科目全称} {考试类型} 考试范围"` — 获取考试范围说明
4. `"{科目全称} {考试类型} 重点"` — 获取高频考点
5. `"{科目全称} 推荐教材 目录"` — 获取主流教材目录
6. `"{科目全称} 教材 目录"` — 获取教材目录

**结果处理**：
1. 提取搜索结果中的章节标题和知识点列表
2. 与模型生成的默认结构交叉验证（合并、去重、重排序）
3. 标记来自 WebSearch 的章节（若与默认结构冲突，优先 WebSearch——更接近真实大纲）
4. 记录搜索来源到 `ai_generation_meta.websearch_queries`
5. 若所有搜索结果均无有效内容，回退到默认知识库生成

#### B.1.3 ISBN / 教材信息精准匹配（v1.5.1 新增）

用户提供 ISBN 码或精确的教材信息（书名+作者+出版社+版次），系统搜索公开目录/图书馆 OPAC/出版社页面，提取真实章节目录作为大纲骨架。

**适用场景**：
- 用户手上有教材（或知道是哪本教材），但不想/不方便上传 PDF
- 希望学习内容与特定教材对齐，但接受模型填充知识点内容
- 学校指定了参考教材，想按教材顺序学习

**信息收集**：

```
📚 教材信息

请提供以下任一信息，我帮你找到教材的真实目录：

方式一（最精准）：
  ISBN：{user input，如 978-7-121-xxxxx}

方式二（也很快）：
  书名：{user input}
  作者：{user input，可选}
  出版社：{user input，可选}
  版次：{user input，可选，如"第5版"}
```

**查询策略**（按优先级）：

| 优先级 | 查询 | 目标源 |
|--------|------|--------|
| 1 | `"{ISBN}"` 精确搜索 | 出版社官网、图书馆 OPAC |
| 2 | `"{书名} {作者} 目录"` | 出版社页面、豆瓣、Google Books |
| 3 | `"{书名} {版次} 目录"` | 出版社页面、图书馆目录 |
| 4 | `"{书名} 目录 site:edu.cn"` | 大学图书馆 OPAC |
| 5 | `"{书名} table of contents"` | Google Books、Amazon |
| 6 | `"{书名} {作者} 章节目录"` | 百度百科、豆瓣 |

**结果处理**：
1. **提取目录**：从搜索结果中提取章节标题列表。来源可信度排序：出版社官网 > 图书馆 OPAC > Google Books > 豆瓣 > 其他
2. **结构化**：将提取的章节标题整理为 `{num, title}` 格式
3. **完整性检查**：
   - 若提取到全部章节（通常 6-12 章）：直接使用此目录作为大纲
   - 若只提取到部分章节（如仅前 3 章）：补充说明"已匹配到前 3 章，后续章节由 AI 基于训练知识补齐"，在展示时标注来源
   - 若完全无法提取：回退到 B.1.1 默认知识库生成
4. **知识点标识**：每章仅根据目录中的核心知识点关键词做初步标注（不做深度知识点生成——深度生成在 B.3 按章延迟执行）
5. **记录来源**：写入 `ai_generation_meta.isbn_lookup_meta`

**输出标注**：展示大纲时标注每章来源：

```
📋 已匹配到教材 "{书名}"（{作者}，{出版社}，{版次}）的章节目录

内容来源：公开目录提取（{来源URL简述}）

| 章节 | 标题 | 来源 | 核心知识点 |
|------|------|------|-----------|
| 第1章 | {标题} | 📖 目录提取 | {知识点1}、{知识点2}... |
| 第2章 | {标题} | 📖 目录提取 | {知识点1}、{知识点2}... |
| 第3章 | {标题} | 📖 目录提取 | {知识点1}、{知识点2}... |
| 第4章 | {标题} | 🤖 AI 补全 | {知识点1}、{知识点2}... |
```

**法律与伦理约束**：
- ✅ 目录/知识点列表是事实信息，不受版权保护
- ✅ 所有搜索来源必须是公开网页（出版社官网、图书馆 OPAC、Google Books 预览、豆瓣）
- ❌ 不得尝试下载或抓取整本 PDF
- ❌ 不得访问需要付费/登录的站点（百度文库会员区、超星等）
- ❌ 若搜索结果中仅出现盗版站点链接，忽略该结果
- 在展示大纲时注明内容来源 URL

### B.2 阶段二：用户确认与编辑

**必须在生成后展示章节目录，等待用户确认。不得跳过此步骤。**

展示格式：

```
📋 已生成 "{科目}" 的章节目录（共 {N} 章）

考试类型：{exam_type}
内容来源：{AI 训练知识 / AI + WebSearch 交叉验证}

| 章节 | 标题 | 核心知识点 |
|------|------|-----------|
| 第1章 | {标题} | {知识点1}、{知识点2}、{知识点3}... |
| 第2章 | {标题} | {知识点1}、{知识点2}、{知识点3}... |
| ... | ... | ... |

你可以：
[A] 确认使用此目录，开始学习 ✓
[B] 手动编辑 — 告诉我需要增删改哪些章节
[C] 使用 WebSearch 增强 — 搜索真实教学大纲补充/校验（仅在未使用 WebSearch 时显示）
```

> 重要：若 `syllabus_source = "generated"`（未使用 WebSearch），选项 [C] 始终可用

**用户选择 [B] 后的交互**：
- 用户可以输入："增加一章 信道编码"、"删除第3章"、"把第5章和第6章合并"、"把量子通信改为经典通信"
- 修改后重新展示目录，再次确认
- 记录修改到 `ai_generation_meta.user_edits`

**知识盲区回退**：若模型无法生成合理的章节结构（过于冷门的科目），回退提示：

```
⚠️ 对于 "{科目}"，我的训练知识不足以生成准确的章节结构。

建议：
[A] 如果你有教材 PDF，请上传，我从教材提取章节
[B] 请手动告诉我有哪些章节（输入章节名称列表）
[C] 尝试用 WebSearch 搜索 "{科目} 课程大纲"
```

### B.3 阶段三：逐章知识点生成（延迟加载）

用户确认章节结构后，知识点**按章按需生成**——学哪章生哪章，保持逐章学习循环的隔离性。

当进入某章学习时（Step 3），执行以下流程：

1. **生成该章完整知识点列表**：
   - 模型基于训练知识生成该章的完整知识点列表
   - 每个知识点包含：

   ```
   - id: "ch{N}_{key}"（如 ch1_comm_model）
   - title: 知识点名称
   - stars: 重要性（3/2/1）
   - concepts: 核心概念列表
   - definition: 概念定义（1-2 句）
   - key_formulas: 关键公式（LaTeX 格式）
   - typical_example: 典型例题思路（1-2 句，可选）
   - pitfalls: 常见误区/易混淆点
   ```

2. **重要性判定规则**（与路径 A 一致）：
   - ⭐⭐⭐：该科目的核心理论 / 有推导 / 是后续前置 / 考试必考
   - ⭐⭐：有定义的独立概念 / 有例题 / 常在习题中出现
   - ⭐：背景介绍 / 拓展阅读 / 进阶内容
   - 📌：常见误区 / 易混淆概念对

3. **可选 WebSearch 增强**（仅该章）：
   - 若用户在 B.1 阶段选择了 WebSearch，或中途要求增强
   - 搜索：`"{章节标题} {核心知识点} 公式"`、`"{章节标题} 经典例题"`
   - 搜索结果补充到知识点的 `key_formulas` 和 `typical_example` 中

4. **输出格式与路径 A 完全一致**：下游 agent 无需感知来源差异。

### B.4 输出格式（内部表示，与路径 A 一致）

```
章节: {num: 1, title: "绪论"}
知识节点:
  - id: "ch1_comm_model", title: "通信系统模型", stars: 3,
    concepts: ["信源", "发送设备", "信道", "接收设备", "信宿"],
    definition: "通信系统由信源、发送设备、信道、接收设备和信宿五部分组成...",
    key_formulas: [],
    pitfalls: ["混淆模拟通信与数字通信的判断标准——关键是消息是否离散"]

  - id: "ch1_entropy", title: "信息熵", stars: 3,
    definition: "离散信源的平均信息量，表示信源每符号的平均不确定性",
    key_formulas: ["H(X) = -\\sum_{i=1}^{n} p(x_i) \\log_2 p(x_i)"],
    pitfalls: ["熵与自信息量的区别——熵是全局平均，自信息量是单个符号"],
    typical_example: "已知信源概率分布，计算信息熵——带入公式直接求和"
```

### B.5 质量约束

1. **章节数限制**：6-12 章。少于 6 章说明科目过于简单或划分太粗，多于 12 章应合并相近章节
2. **每章知识点**：⭐⭐⭐ 2-5 个，⭐⭐ 3-8 个，⭐ 不限
3. **公式必须 LaTeX**：行内 `$...$`，行间 `$$...$$`
4. **不生成禁词**：不得出现"当然""显然""易证""众所周知"等词汇
5. **优先使用国内教材术语**：如"傅里叶变换"而非"Fourier Transform"，"信道"而非"Channel"

## 路径 M：Markdown 提取（v1.6 新增）

### 前置条件

- `progress.json` 中 `source_type = "markdown"`
- 用户提供的 Markdown 文件为预提取的结构化文本（含章节标题和 LaTeX 公式）

### M.1 章节结构识别

从 Markdown 标题层级推断章节目录：

```
# 书名/科目名           → 教材名称
## 第X章 / Chapter X    → 章边界（若标题不含"第X章"，按顺序编号）
### 第Y节 / 小节名       → 节边界
#### ...                → 子节（可选）
```

**标题模式匹配**（按优先级）：
1. `## 第\d+章` / `## Chapter \d+` → 精确章边界
2. `## 1.` / `## 一、` → 数字/中文序号章边界
3. 若以上均不匹配 → 所有 `##` 标题按出现顺序编号为第 1, 2, 3...章

**无标题回退**：若 Markdown 无 `##`/`###` 标题层级，按以下规则推断：
- 分隔线 `---` 作为章边界
- 连续 3 行以上的空行作为节边界
- 以 `**第X章**` 或 `**Chapter**` 开头的加粗文本视为章标题

### M.2 知识点提取

从 Markdown 正文中识别知识节点，扫描模式：

| 模式 | 识别目标 | 优先级 |
|------|---------|--------|
| `**定义**` / `> **定义**` 块引用 | 概念定义 | 高 |
| `**定理**` / `**性质**` / `**推论**` | 定理/性质 | 高 |
| `$...$` / `$$...$$` | 公式（已为 LaTeX，直接使用） | 高 |
| `**注意**` / `**⚠️**` / `> ⚠️` | 易混淆点/误区 → 📌 | 中 |
| `**例题**` / `**示例**` / `**例\d+**` | 典型例题思路 | 中 |
| 列表项中出现的加粗术语 | 核心概念候选 | 低 |

**公式处理**：
- Markdown 中公式已为 LaTeX 格式，**直接提取，无需转换**
- 行内公式 `$...$` → 提取为 `key_formulas` 成员
- 行间公式 `$$...$$` → 提取为 `key_formulas` 成员
- 含 `\label{}` 或编号的公式 → 标记为关键公式

**知识点聚合**：
- 相邻的定义 + 公式 + 例题 → 合并为一个知识节点
- 独立出现的定理 → 单独成为一个知识节点
- 同一 `###` 节下的内容 → 归属于该节

### M.3 重要性判定

与路径 A 规则完全一致：

```
⭐⭐⭐：在章节小结中出现 / 有完整推导过程 / 明确标注为后续前置 / 含 \label 的关键公式
⭐⭐  ：有定义块 / 有例题 / 在多处被引用
⭐    ：背景介绍 / 拓展阅读 / 仅出现一次且无推导
📌   ：标注「注意」「常见误区」「⚠️」「易混淆」的内容
```

### M.4 等级过滤

与路径 A 规则完全一致：

```
L1: 仅 ⭐⭐⭐
L2: ⭐⭐⭐ + 前置依赖的 ⭐⭐
L3: ⭐⭐⭐ + 全部 ⭐⭐ + 小结中的 ⭐
L4: 全部
```

### M.5 输出格式

与路径 A 完全一致，下游 agent 无需感知来源差异：

```
章节: {num: 1, title: "绪论"}
知识节点:
  - id: "ch1_comm_model", title: "通信系统模型", stars: 3,
    concepts: ["信源", "发送设备", "信道", "接收设备", "信宿"],
    definition: "...",
    key_formulas: [],
    pitfalls: ["混淆模拟通信与数字通信的判断标准"]
```

### M.6 与路径 A 的差异

| 维度 | 路径 A (PDF/PPT) | 路径 M (Markdown) |
|------|-----------------|-------------------|
| 文本提取 | pymupdf / tesseract OCR | 直接读取（已是纯文本） |
| 公式处理 | OCR → LaTeX 转换 | 直接提取（已是 LaTeX） |
| 章节识别 | 书签 + 字体大小推断 | 标题层级解析 |
| 表格处理 | 需 OCR 识别 | Markdown 原生表格 |
| 图片/图表 | 需截图 + OCR | 保留 `<img>` URL（PaddleOCR 云存储链接），需筛选重要图片嵌入 |
| 处理速度 | 较慢（含 OCR） | 极快（纯文本解析） |
| 精准度 | 取决于 PDF 质量 | 取决于 Markdown 转换质量 |

### M.7 图片处理（v1.6 新增）

Markdown 源文件中保留了 PaddleOCR 转换时提取的图片链接（`<img src="...">` 标签，URL 含有时效性授权 token）。

**筛选规则**：

| 优先级 | 条件 | 示例 |
|--------|------|------|
| **必须嵌入** | 正文中有"如图 X.X 所示"明确引用的图 | 图 2.3 定义 $X(t,\xi)$ 的示意图 |
| **建议嵌入** | 概念关系图、结构图、关键波形图 | 一阶密度函数示意图 |
| **不嵌入** | 出版社 logo、装饰图、页面截图装饰 | 封面、CIP 页的印章图 |

**嵌入方式**：
- HTML 模式：`<figure>` + `<img>` + `<figcaption>` 结构，居中展示，附带原文图号
- Markdown 模式：`![描述](URL)` 标准语法 + 斜体图注
- 每个嵌入图片附带文本描述（`alt` 属性），作为 URL 过期或加载失败的回退

**图片 URL 处理**：
- 直接使用 PaddleOCR 返回的原始 URL（已验证 200 可访问）
- 不下载、不转存、不 base64 内嵌（保持文件轻量）
- URL 过期后图片将静默失效，`alt` 文本作为回退说明
