# 铁律（IRON RULES）与反模式

## 测评规则

| 编号 | 规则 |
|------|------|
| **R-ASSESS-1** | 摸底测评题必须从全书 ⭐⭐⭐ 知识点中抽取 |
| **R-ASSESS-2** | 章节小测只能考最近一次学习手册中的内容，不能超纲 |
| **R-ASSESS-3** | 选择题自动评判，简答题 LLM 评判（按准确性/逻辑/公式/表达四维评分） |
| **R-ASSESS-4** | 每次测验后必须生成错题集，包含解析 |

## 自适应规则

| 编号 | 规则 |
|------|------|
| **R-ADAPT-1** | 学习手册的知识点覆盖严格按当前等级过滤 |
| **R-ADAPT-2** | 重学本章时，等级基于上次小测成绩更新 |
| **R-ADAPT-3** | 同一章节重学次数不设上限，直到用户满意 |
| **R-ADAPT-4** | 每章独立跟踪等级（per_chapter_level），不再使用全局等级（v1.2 新增） |

## 过渡语言规则（v1.4 新增）

测评、预检、小测等评分环节结束后，**禁止**使用机械式过渡语。必须根据分数给出自然反馈。

| 分数段 | 反馈基调 | 示例（懒羊羊风格） |
|--------|---------|-------------------|
| 90-100 | 惊喜 + 真诚夸奖 + 解释会教更深 | "全对？！你比本羊还聪明！既然底子这么好，本羊给你讲全一点，别浪费了——" |
| 75-89 | 肯定 + 指出亮点 + 解释覆盖策略 | "不错嘛，87 分！大部分都掌握了。本羊给你补几个还没吃透的点——" |
| 60-74 | 鼓励 + 找具体薄弱点 + 解释重点聚焦 | "还行还行，60 多分说明基础有的。这块本羊帮你抓重点，别的不急——" |
| 0-59 | 不贬低 + 找亮点 + 解释从基础开始 | "没关系，这块确实还没学过嘛。从最基础的来，本羊陪你慢慢啃——" |

**核心原则**：
- 分数好 → 夸具体行为（"信息熵那题秒答，说明你公式记得很牢"），不用空洞的"很好"
- 分数差 → 归因于客观（"这块教材确实绕"），不归因于学生能力
- 永远解释等级意味着什么（"为你准备了更全的知识点"而不是"L3 模式"）
- 过渡语必须符合当前 IP 角色的语气

## 内容规则

| 编号 | 规则 |
|------|------|
| **R-CON-1** | 学习手册必须包含「学习目标」和「自检问题」段 |
| **R-CON-2** | 错题集必须包含「错误类型」和「学习建议」列 |
| **R-CON-3** | 公式使用 LaTeX 语法 |
| **R-CON-4** | 学习手册/讲解内容必须包含可视化图表：概念关系图用 Mermaid（graph/flowchart），波形/频谱/叠加图用 `scripts/plot_signals.py` 生成 PNG（暗色主题，与 HTML 配色统一），禁止使用 ASCII 字符画代替真实图表（v1.2 新增，v1.7.0 修订） |
| **R-CON-5** | AI 生成的知识点必须使用与 PDF 提取相同的内部表示格式（id/title/stars/concepts/key_formulas/pitfalls）（v1.5 新增） |
| **R-CON-6** | 学习手册须逐知识点展开：每个 ⭐⭐⭐ 点必须有独立小节（定义+公式+角色洞见），不得压成纯表格或 bullet list。HTML 模式下每张 card 内至少包含一段公式块或一张图（v1.6 新增） |
| **R-CON-7** | 单文件双 Tab 模式下，学习手册 Tab 的内容密度不得因合并且降低。生成顺序：先保证手册质量达标（六段式完整、每 ⭐⭐⭐ 独立 h3、公式块+blockquote 齐全），再叠加小测 Tab。文件行数不作为约束条件，500+ 行可接受（v1.6.1 新增） |
| **R-CON-8** | 例题必须包含完整的「题目」段和「解」段，推导步骤不可省略。每个例题须有：① 明确的问题陈述（"求..."）② 分步推导（关键公式展开）③ 最终答案。禁止将题目+推导+答案压缩为一句摘要（v1.6.1 新增） |
| **R-CON-9** | 每个 Session/章节讲解必须包含至少一张 Mermaid 概念关系图（graph LR），展示本节知识点之间的逻辑关系（v1.7.0 新增） |
| **R-CON-10** | 每个 Session/章节讲解必须包含一张 Mermaid 全景路线图（flowchart TD），标注当前进度位置，让学习者知道"我在哪、下一步是什么"（v1.7.0 新增） |
| **R-CON-11** | `render_mode = "html"` 时，Mermaid 须在 `<head>` 加载 CDN 并初始化暗色主题配置（`primaryColor:#1e3a5f`、`lineColor:#94a3b8`），确保与页面暗色主题统一（v1.7.0 新增） |
| **R-CON-12** | Mermaid 图表和 HTML 正文中**禁止使用 Unicode 特殊符号**（如 ↔ ⇔ 🕐 🎵 等），原因：中文字体（STHeiti/PingFang）可能不含这些字形导致静默不渲染。替代方案：Mermaid 标签用 ASCII（`--` 代替 `↔`），HTML 正文用 MathJax（`$\longleftrightarrow$` 代替 `↔`，`$\delta$` 代替 `δ`）（v1.7.0 新增） |
| **R-CON-13** | `render_mode = "html"` 时，若公式中使用 `\xleftrightarrow`、`\xrightarrow` 等可扩展箭头命令，MathJax 配置**必须**加载 `extpfeil` 扩展：`packages: {'[+]': ['extpfeil']}` + `loader: { load: ['[tex]/extpfeil'] }`，否则箭头不渲染（v1.7.0 新增） |

## 复习规则（v1.3 新增）

| 编号 | 规则 |
|------|------|
| **R-REVIEW-1** | Step 0 必须运行 `review_calc.py check`，若有过期项则提示复习 |
| **R-REVIEW-2** | 每章小测完成后，必须用 `review_calc.py init` 将该章 ⭐⭐⭐ 知识点写入 knowledge_tracking |
| **R-REVIEW-3** | 复习交互时，quality 由用户答题结果映射：全对=5，部分对=3，全错=1 |
| **R-REVIEW-4** | 复习完成后必须运行 `review_calc.py update` 更新 SM-2 参数 |
| **R-REVIEW-5** | 复习题必须从该知识点对应的学习手册中出题，不超纲 |
| **R-REVIEW-6** | 错题知识点（is_error=true）的初始 EF 降低为 2.0，优先排入复习队列 |
| **R-REVIEW-7** | 每章小测评卷完成后，2 步强制执行：① `review_calc.py init` 写入 ⭐⭐⭐ 知识点（v1.6.1 强调：禁止跳过）② 下次进入任意章节学习前，必须运行 `review_calc.py check` 检测到期项（v1.6.1 新增） |

| 编号 | 规则 |
|------|------|
| **R-LEARN-1** | Step 0 必须根据输入自动检测模式：上传文件 → exam，纯关键词 → learn。不确定时主动询问用户 |

## 边界探测规则（v1.7.0 新增）

| 编号 | 规则 |
|------|------|
| **R-PROBE-1** | 学习模式下每完成 3 个 session/主题，必须自动触发一轮边界探测 |
| **R-PROBE-2** | 边界探测固定 7 题，全部为简答题（禁止 MCQ），一次一题 |
| **R-PROBE-3** | 难度分布：Q1 低于当前级 → Q2-Q3 当前级 → Q4 试探 → Q5 探测 → Q6-Q7 动态调整 |
| **R-PROBE-4** | Q4 得分 ≥ 7 时 Q5 加难；Q4 得分 ≤ 4 时 Q5 降回当前级；Q5 答不上来则 Q6-Q7 确认边界 |
| **R-PROBE-5** | 每题必须打分（0-10），指出正确部分、漏洞，并针对性补讲 |
| **R-PROBE-6** | 探测结束后必须生成探测报告：得分、理解边界、级别调整、漏洞列表 |
| **R-PROBE-7** | 级别升级条件：Q1-Q3 均 ≥ 7 且 Q4-Q5 均 ≥ 6；降级条件：Q1-Q3 有 ≤ 4 |
| **R-PROBE-8** | 答错题目必须触发费曼循环补讲（最多 3 轮），不额外加补课 session |
| **R-PROBE-9** | 探测中的答题结果必须写入 knowledge_tracking（答对 quality 4-5，答错 quality 1-2） |
| **R-PROBE-10** | 探测后生成本轮 3 个 session 的合并速查表，并运行 `review_calc.py init` |
| **R-PROBE-11** | 探测历史必须写入 `progress.json` 的 `boundary_probes` 数组 |
| **R-PROBE-12** | 探测结束后，后续 session 必须加重本轮暴露的薄弱内容（不另加补课 session） |

## 反模式

| # | 反模式 | 正确做法 |
|---|--------|---------|
| 1 | 摸底测评出偏僻题 | 只从 ⭐⭐⭐ 核心知识点出题 |
| 2 | 章节小测考没学过的内容 | 只能从最近一次学习手册中选材 |
| 3 | 重学本章时照搬上次的学习手册 | 根据新等级重新过滤知识点 |
| 4 | 所有学生用同一份学习手册 | 按等级过滤，不同水平不同覆盖 |
| 5 | 不记录学习历史 | 每次学习后更新 progress.json |
| 6 | 错题集只有答案没有解析 | 每道错题必须有解析 + 学习建议 |
| 7 | 最终检测不询问往年试卷 | Step 6 必须先询问 |
| 8 | 所有学生统一教学风格 | Step 0 必须先选 Persona |
| 9 | 速成模式给学生讲长篇证明 | Persona D 只讲结论和口诀 |
| 10 | 考研模式只做基础题 | Persona E 必须包含拔高和真题变体 |
| 11 | 重学时不引用上次错题 | 重学手册必须注入错题集的易错点（v1.2 新增） |
| 12 | 所有章节使用同一全局等级 | 每章独立跟踪等级（v1.2 新增） |
| 13 | 不追踪学习时间戳 | 每次学习/复习后写入 learned_at/last_reviewed（v1.3 新增） |
| 14 | 复习环节让 Claude 手动算日期 | 用 review_calc.py 做确定性计算（v1.3 新增） |
| 15 | 小测后不初始化 knowledge_tracking | Step 5 必须调用 review_calc.py init（v1.3 新增） |
